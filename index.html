Dota 2 Chat Web Application

Images

Description

Chat Web application that features account registration and chat room creation using NodeJS Express on the server side, RESTful API and sockets for client-server communication, HTML5+BOOTSTRAP+JQuery for the client side.

Visit Page

Heroku Git URL: https://git.heroku.com/jediquinmidtermproject.git

Reflection

When it comes to presenting something that is visually appealing, this is my best work. This is the first major web application project that I’ve done where I set up a server and used RESTful API, web sockets, and bootstrap. I have applied all the things that I’ve learned from the web development courses I took at BCIT and the things that I taught myself which I think is very useful in web development.

The project was not that hard, but it would test how well you know how RESTful API and sockets work. I first implemented the functionalities and created a simple page. As I slowly develop this project, the vision of how I wanted it to look like became clearer. Page by page, element by element, I started structing things. I had minor problems with the layout because bootstrap was not really taught to us, but a little Googling and research fixed the problem.

As I was getting close to completion, I've added more features to it than what our instructor required. For me, this was not only just a course project to satisfy requirements, this became a project I can show to employers.

I had so much fun in making this project. I’m very happy on how it turned out and very proud of this work.

PCAP Miner

Images

Description
Java program using JavaFX library for User Interface that process PCAP Files and shows the following:
  • Packet count
  • Average packet per sec
  • Average packet size
  • Conversation statistics for the following protocols:
    • Ethernet
    • TCP & UDP
    • IPv4 & IPv6Protocols
Github Page

Git URL: https://github.com/jedski01/2515-Project-PCAP_Miner.git

Reflection

While the chat web app is my visually presentable work, when it comes to functionality, our PCAP miner is my best work. My groupmate and I made a program using Java/JavaFX that reads PCAP files (the Wireshark capture files) and presents statistics of that capture file to the user. We basically implemented the Conversation statistic found on the original Wireshark program with some minor additional fields such as TTL and packet retransmissions.

We ran into a few problems with JavaFX, as it was a new thing to us. At first, we had a very poorly made interface. IDE would hang up on us because the component tree was too deep and there were too many components in one scene. Fortunately, our instructor gave us some JavaFX design patterns that turned out to be a significant help to us. One of those patterns were multiple scenes in one stage. Because of that, we were able to refactor our code and separate scenes into multiple files. This made things easier to modify as it was not in one big file, and at the same time, the IDE did not hang up on us.

Another problem we had is with the library we used to parse the PCAP files. The API documentation we’re not sufficient and some methods we’re not properly explained. And the same time, there were very few examples/code snippets online that uses the library. I feel like this limited our features, for example, we initially wanted to add statistics for wireless packets but it turns out the library did not support wireless packets at that time.

Our instructor also taught us about code smells while making the project. We were then tasked to refactor our code. I learned so much from doing this project, and I feel like it made me a better programmer.
